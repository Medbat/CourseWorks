* Объектно-ориентированная библиотека для проверки и преобразований ZIP-архивов

  ZIP-архивы широко используются. Например, программы на Java обычно
  распространяются в виде .jar-файлы, которые являются
  ZIP-архивами. .docx-файлы, .xlsx и т.п. являются
  ZIP-архивами. Полезно проверять, что структура ZIP-архива не
  повреждена, восстанавливать её и т.п.

  Здесь не предполагается работать со сжатием файлов, а только со
  структурой архива.

  Что можно делать:
  - преобразовывать структуру архива в текстовое описание, которое
    можно читать человеку и, возможно, править;
  - восстанавливать архив обратно из текстового описания;
  - создавать архив из каталога файлов (без сжатия);
  - проверять непротиворечивость структуры архива:
    - не накладываются ли файлы внутри архива согласно оглавлению;
    - информация о файлах внутри архива записана в двух местах -- в
      основной части архива и в оглавлении (в конце файла) --
      совпадает ли информация;
    - не очень ли большой коэффициент сжатия (Zip bomb).
